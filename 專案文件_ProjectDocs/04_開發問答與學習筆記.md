# 04 開發問答與學習筆記

以 Q&A 格式整理開發過程中的「關鍵問題」與「解決方案」，作為未來的防呆筆記與學習紀錄。

---

## Q1: 發現電腦卡頓時，如何找出並關閉佔用大量資源的「效能小偷」？
**狀況描述**：在執行系統資源監控 (Phase 2) 時，發現 `M365Copilot` 與 `msedgewebview2` 佔用了極高的 CPU 與記憶體。

**解決方案與學習**：
1. **M365Copilot (背景 AI)**：
   * **治標**：打開「工作管理員」，找到該程序並點擊「結束工作」。
   * **治本**：前往 Windows「設定」>「應用程式」>「啟動」，將 Copilot 或 M365 相關項目設為「關閉」。
2. **msedgewebview2 (Edge 網頁渲染底層)**：
   * **⚠️ 觀念釐清**：這不是單純的瀏覽器，而是許多 Windows 應用程式（如 Teams、天氣小工具）依賴的底層畫布，**絕對不能強制刪除**。
   * **治本**：打開「工作管理員」的「啟動應用程式」分頁，將 **Microsoft Edge** 設為「停用」。另外也可關閉下方工具列的「小工具 (Widgets)」，能大幅減少背景生成的數量。

## Q2: 使用 GitHub CLI (`gh`) 登入時，為什麼終端機會卡住或顯示尚未登入？
**狀況描述**：執行了 `gh auth login`，終端機顯示了代碼卻沒反應，或是貼上下一個指令時系統報錯 `To get started with GitHub CLI, please run: gh auth login`。

**解決方案與學習**：
* **⚠️ 觀念釐清**：GitHub 的登入**不是**在終端機內打密碼，而是必須透過「**網頁瀏覽器**」來完成授權。
* **正確的標準流程**：
  1. 在終端機執行 `gh auth login`。
  2. 複製畫面上出現的 8 碼一次性代碼（如 `D1DC-A202`）。
  3. **按下 Enter 鍵**（這會觸發系統打開瀏覽器）。如果沒反應，手動打開瀏覽器前往 `https://github.com/login/device`。
  4. 在網頁中貼上代碼並授權。
  5. 🛑 **關鍵防呆**：授權後，**絕對不能馬上關掉終端機**！必須回到終端機畫面，等待它顯示綠色打勾的 `✓ Logged in as [您的帳號]`。看到這行字，才代表電腦真的拿到了登入鑰匙。

## Q3: 為什麼終端機找不到 `gh` 指令 (無法辨識 'gh' 詞彙)？
**狀況描述**：剛剛才用 Winget 安裝好 GitHub CLI，馬上執行 `gh` 指令卻跳出紅字報錯。

**解決方案與學習**：
* **原因**：Windows 的終端機在開啟的當下，只會讀取「當時」的環境變數。新安裝的軟體路徑還沒被舊的終端機視窗認識。
* **解法**：最簡單的方法就是**把目前的終端機視窗關掉 (打叉)，重新開一個新的**。如果不想重開，可以使用絕對路徑來呼叫程式（例如 `& "C:\Program Files\GitHub CLI\gh.exe"`）。

## Q4: 本機 Git 與 雲端 GitHub 的差別是什麼？「完成了」到底是指哪裡完成？
**狀況描述**：在本地端已經成功執行了 `git init`、`git add` 和 `git commit`，但網頁上還是看不到專案。

**解決方案與學習**：
* **本機端 Git (打包裹)**：執行 `commit` 就像是把行李打包好放在家門口。到這一步為止，檔案是有被保護與紀錄的，但**只存在於您的這台電腦硬碟裡**。
* **雲端 GitHub (上傳雲端)**：必須要有「貨運公司」來收件。執行 `gh repo create ... --push` 就像是呼叫貨運公司把包裹搬上雲端。
* **結論**：必須要在終端機看到 `Pushed commits to https://github.com/...`，並且在瀏覽器重新整理 GitHub 網頁有看到檔案，才算是真正完成了 100% 的跨裝置雲端備份！
