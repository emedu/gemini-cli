# 02 執行報告

## 專案進度總結
* **目前已完成的功能**：
  已完整執行「Gemini x Windows 系統診斷專案計畫書」的四大階段檢測，包含：系統體質與環境掃描、資源瓶頸即時監控、穩定性與錯誤日誌分析、啟動項與服務檢查。

## 測試與診斷結果 (Findings)
1. **硬體體質**：
   * 主機板型號為 Z390 AORUS ELITE，處理器為 Intel Core i9-9900K，具備強大的多工處理能力。
   * C、D、E 槽磁碟空間皆十分充足，硬碟閒置率高達 99.8% 以上，無 I/O 瓶頸。
2. **效能瓶頸 (資源怪獸)**：
   * **M365Copilot** 與 **msedgewebview2** (Microsoft Edge 背景引擎) 是最大的系統資源消耗來源。
3. **穩定性與系統日誌**：
   * 系統底層健康，無致命硬體錯誤。
   * 主要錯誤為 `0x80073d02` (Windows Update 嘗試更新使用中的應用程式失敗)，主因是 Edge WebView2 或 Copilot 等程式常駐佔用導致。
4. **啟動項與服務**：
   * 發現多個非必要的開機自啟動程式，包含：Microsoft Edge、Google Drive、Epson 印表機監控程式 (EPSDNMON)、以及 OneNote 快捷鍵。

## 遇到的主要困難與解決方式 (優化處方籤)
* **問題**：系統因過多的背景程式與網頁渲染引擎常駐，造成 CPU 與記憶體負擔。
* **解決方案**：
  1. **已處理**：在「工作管理員」的啟動分頁中，將 `msedge.exe` (Microsoft Edge) 設為停用。
  2. **建議處置**：停用非必要的開機啟動項 (如 Epson 監控程式、少用的雲端同步軟體)。
  3. **背景 AI 降載**：若資源仍吃緊，可從設定中停用 M365Copilot，或在工作列關閉「小工具 (Widgets)」以減少 WebView2 的生成。
