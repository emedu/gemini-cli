# 03 技術手冊：快速啟動與應用指南

本文件作為教學指南，將告訴您如何透過終端機「開啟」這個 Gemini CLI 專案，以及您可以利用它來做什麼。

---

## 🚀 第一部分：如何用終端機開啟 Gemini CLI？

目前我們的專案資料夾已經在硬碟中準備就緒。要「開啟」或「進入」它，您只需要熟悉兩個最基礎的終端機指令：

### 步驟 1：進入專案資料夾 (開啟它)
每次打開 PowerShell 終端機時，請先輸入這行指令並按 Enter：
```powershell
cd D:\projects\gemini-cli
```
*(💡 `cd` 的意思是 Change Directory，代表您把終端機的指揮中心「走進」了這個資料夾。)*

### 步驟 2：開啟編輯器 (開發它)
如果您想看到資料夾裡面的檔案並進行編輯，最快的方法是在該資料夾中輸入：
```powershell
explorer .
```
*(這會幫您彈出 Windows 檔案總管的視窗。)*

> **未來進階用法**：如果您之後安裝了像 VS Code 這樣的寫程式專用軟體，您只要在終端機輸入 `code .`，就能瞬間把整個專案在專業編輯器中打開！

---

## 🎯 第二部分：我可以利用這個專案做什麼？

目前，我們在這個專案裡建立了完整的「**Windows 系統診斷腳本與報告**」。但既然專案名稱叫做 **Gemini CLI** (Command Line Interface，命令列工具)，它的潛力遠不止於此！

以下是這個專案現在可以做，以及未來能夠擴充的 3 大階段：

### 🟢 階段一：作為「系統維護寶典」 (目前的狀態)
您可以隨時用終端機 `cd` 進來查看 `專案文件_ProjectDocs`，裡面記錄了我們如何揪出效能小偷、如何關閉無用啟動項的完整紀錄。如果以後電腦又變卡，只要打開 `01_執行計畫.md` 複製裡面的 PowerShell 指令去跑，就能立刻自己當電腦醫生。

### 🟡 階段二：打造自動化檢測腳本 (下一步)
既然我們已經把那些好用的 PowerShell 檢測指令找出來了，我們可以把它們寫成一個 `.bat` 或 `.ps1` 執行檔！
未來您只要在終端機輸入一行指令（例如：`./check-system.ps1`），電腦就會自動把 CPU、硬碟、開機啟動項掃描一次，直接印出一份報告給您，不用再一行一行複製貼上。

### 🔴 階段三：串接 Gemini AI，變成真正的「Gemini CLI」 (終極目標)
這是一個非常酷的開發方向！由於我們已經初始化了 Node.js (`package.json`)，我們可以寫一隻真正的程式：
1. 程式自動執行 PowerShell 取得電腦數據。
2. 程式自動透過 API 把數據丟給 Gemini AI。
3. Gemini AI 自動回傳「中文的電腦健檢建議」，並直接顯示在您的終端機上！

到那個時候，您只要在終端機輸入類似 `gemini-cli diagnose`，一個擁有 AI 大腦的電腦醫生就會直接在黑底白字的終端機裡為您看診！
