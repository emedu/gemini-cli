# 01 執行計畫

## 專案目標與背景
Gemini x Windows 系統診斷專案計畫書

這是一個結構化的診斷流程。請依照階段順序執行指令，並將結果回傳給我進行分析。

📋 專案目標

透過 PowerShell 指令抓取系統核心數據，利用 AI 分析找出電腦卡頓、效能下降或異常報錯的根源，並提供量身定制的優化建議。

## 階段 1：系統體質與環境掃描 (System Profile)

目的： 確認你的硬體基礎與作業系統狀態。

⌨️ 請執行以下 PowerShell 指令：

```powershell
# 1. 取得系統基本資訊 (型號、記憶體、處理器)
Get-ComputerInfo | Select-Object CsModel, OsName, OsVersion, CsProcessors, WindowsProductName

# 2. 檢查磁碟剩餘空間
Get-Volume | Select-Object DriveLetter, FriendlyName, SizeRemaining, Size | Where-Object {$_.DriveLetter -ne $null}
```

## 階段 2：資源瓶頸即時監控 (Resource Bottleneck)

目的： 找出到底是誰在吃掉你的效能（CPU、記憶體或硬碟）。

⌨️ 請執行以下 PowerShell 指令：

```powershell
# 1. 列出消耗 CPU 最多的前 10 個程式
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10 -Property Name, CPU, WorkingSet

# 2. 檢查硬碟活動率 (如果數值很低表示硬碟很閒，很高表示硬碟忙不過來)
Get-Counter '\LogicalDisk(*)\% Idle Time'
```

## 階段 3：穩定性與錯誤日誌 (Stability & Errors)

目的： 檢查系統是否有潛在的檔案損壞或硬體警告。

⌨️ 請執行以下 PowerShell 指令：

```powershell
# 1. 檢查系統檔案完整性 (僅掃描，不修復)
sfc /verifyonly

# 2. 抓取最近 10 筆系統錯誤日誌
Get-EventLog -LogName System -EntryType Error -Newest 10 | Select-Object TimeGenerated, Source, Message
```

## 階段 4：啟動項與服務檢查 (Startup & Services)

目的： 找出哪些程式在開機時偷偷啟動，拖慢了你的反應速度。

⌨️ 請執行以下 PowerShell 指令：

```powershell
# 列出所有自動啟動的程式與路徑
Get-CimInstance Win32_StartupCommand | Select-Object Caption, Command, User
```

## 📈 如何與 Gemini 協作診斷？

1. **複製指令**： 從上面的階段中複製代碼。
2. **執行**： 在 Windows 中按下 Win + X，選擇「終端機 (管理員)」或「PowerShell」。
3. **回傳**： 將執行的結果貼回對話框。
4. **分析**： 我會針對每一階段的數據給出診斷結果，並引導你進入下一階段。
